# 🔄 Daily GitHub Update Workflow

## Quick Commands

### Check Status
```bash
git status
```

### Add All Changes
```bash
git add .
```

### Commit Changes
```bash
git commit -m "Description of what you changed"
```

### Push to GitHub
```bash
git push
```

---

## One-Line Update (All at Once)
```bash
git add . && git commit -m "Daily update" && git push
```

---

## Common Update Messages

### Feature Updates
```bash
git commit -m "Added new feature: [feature name]"
git commit -m "Improved: [what you improved]"
git commit -m "Fixed: [what you fixed]"
```

### Documentation Updates
```bash
git commit -m "Updated README with new instructions"
git commit -m "Added documentation for [feature]"
```

### Bug Fixes
```bash
git commit -m "Fixed bug: [description]"
git commit -m "Resolved issue with [component]"
```

### Data Updates
```bash
git commit -m "Updated PDF documents"
git commit -m "Added new NPS guidelines"
```

---

## What Gets Pushed

### ✅ Always Included:
- Source code changes (app.py, src/)
- New/updated PDF documents
- Documentation updates
- Configuration changes

### ❌ Never Pushed (Protected by .gitignore):
- .env (your API keys)
- vector_store/ (regenerated by users)
- __pycache__/ (Python cache)
- .backups/ (old files)

## Troubleshooting

### If push fails:
```bash
# Pull latest changes first
git pull origin main

# Then push again
git push
```

### If you need to undo last commit:
```bash
git reset --soft HEAD~1
```

### If you want to see commit history:
```bash
git log --oneline
```
